# 🤝 Guía de Contribución

¡Gracias por tu interés en contribuir a **Modelos 2026 – Normalización, Ponderación y Agregación**!

## 📋 Cómo Contribuir

### 1. Reportar Bugs
Si encontraste un error:
- Abre un **Issue** con título descriptivo
- Incluye:
  - Descripción del problema
  - Pasos para reproducir
  - Comportamiento esperado vs. actual
  - Entorno (SO, versión Python, etc.)

### 2. Solicitar Nuevas Funcionalidades
- Abre un **Issue** con etiqueta `enhancement`
- Describe:
  - Qué funcionalidad falta
  - Caso de uso
  - Beneficio esperado

### 3. Enviar Mejoras de Código
1. **Fork** el repositorio
2. Crea una rama: `git checkout -b feature/mi-mejora`
3. Realiza cambios y documenta
4. Commits claros: `git commit -m "Agregar [descripción]"`
5. Push a tu fork: `git push origin feature/mi-mejora`
6. Abre un **Pull Request** con descripción detallada

## 📐 Estándares de Código

### Python
- Seguir **PEP 8**
- Usar nombres descriptivos
- Documentar funciones con docstrings
- Comentar lógica compleja

### Notebooks
- Celdas bien organizadas
- Markdown descriptivo entre celdas
- Salidas limpias (sin errores de ejecución)

### Documentación
- README claro y actualizado
- Ejemplos funcionales
- Explicar fórmulas matemáticas

## 🧪 Pruebas

Antes de enviar PR:
- ✅ Ejecutar el notebook completo sin errores
- ✅ Validar con datos de prueba
- ✅ Verificar compatibilidad con versiones de librerías

## 📝 Commits

Formato recomendado:
```
[Tipo] Descripción breve

Descripción detallada si aplica
- Punto 1
- Punto 2
```

**Tipos:** `feat` | `fix` | `docs` | `refactor` | `test`
